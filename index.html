Craig's Project Page

Many of my projects are open source and hosted here on github. This page is an overview of my work in Haskell.

mutate

My mutate package is an attempt to formalize the relationships between various types of mutable variables such as IORefs, STRefs, and TVars.

The original motivation of this library was to be able to write operators and larger abstractions that would simply do what you expect with different kinds of mutable variables.

This has proven more complicated than I originally anticipated. Types such as IORef, STRef, TVar, and StateVar support similar operations but have subtle semantic differences in regards to exceptions and concurrency. It is an ongoing project.

hurl

hurl is a high-level interface to the Chipmunk physics library that is in early development. It is based on the Hipmunk package, which is a more or less straightforward (yet high-quality) binding to the C library.

The purpose of hurl is to provide a more idiomatic and expressive API than Hipmunk. The most important goal in this regard is the use of functional reactive programming to structure collision handling rather than using callbacks.

drawing 2-D graphics

tooty (pronounced like 2-D) is a simple DSL for 2-dimensional drawing in OpenGL. OpenGL is incredibly stateful, and often painful to use in Haskell. tooty offers a functional-style API for drawing, but does not try to encapsulate the details of OpenGL from the user.

An earlier experiment of mine, draw, was a library similar in spirit to tooty but using SDL for graphics instead of OpenGL. In that library, drawing computations were expressed in terms of a monad Context c a where c constrained the capabilities of the drawing context. It was a fun concept and I might return to it in the future, but next time using type-level records to manage capabilities.

experiments

tree-writer defines Control.Monad.TreeWriter. It is a small specialized version of the WriterT monad transformer used to construct rose trees. It was a neat idea but I've found myself using pipes in all cases where I might have used tree-writer.

weave defines the Weave type. A Weave c a is isormorphic to [Either c a], and can be used as a traversable container of as embedded in the monoid c. For example, Weave Text Image represents a Text value with Images embedded throughout it. I created weave because it encapsulates a general pattern that I recognized in several of my projects, but I'm still determining its utility as a general abstraction.

regular-tile provides the typeclass Tile, a subclass of Eq and Monoid. It represents a regular tile in the 2-D plane (there are three types: triangles, squares, and hexagons). Expressing path-finding algorithms and other logic in terms of the Tile typeclasses enable their use in libraries for boardgame-style computer games that work in any of the three regular tilings.

command line interfaces

I am planning to develop a monadic framework for developing interactive console interfaces. You would use the library to define a tree of possible application states. The tree could be used to generate the command line arguments for your program, and could also be used to generate an "interactive mode" that assists a user who is unfamiliar with the program.

A few of my older projects were experiments working towards this goal:

terminal provides the mtl-style typeclass MonadTerm, which exposes the functionality of a console interface. This is currently implemented in terms of the haskeline package, but it could target other backends as well.

input was a brief experiment with the same concepts as terminal but expressed using operational monads.

save provides a type for managing application resources that should be saved between uses of the application. It uses type-level programming to track the modification of those resources and use that information implicitly for control flow. I wrote it while I was going through a bit of an "over-engineering" phase.

command was a simple monadic parser for structuring application-wide state changes in response to user input on the command line. One goal of the project was to create a composable type representing command line queries of the style (y/n). This was an old project that I abandoned, but I might go back and rewrite it in terms of a standard parser combinator library.
